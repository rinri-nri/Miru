PRIVACY POLICY

プライバシーポリシー

制定日: 2026年7月7日

本プライバシーポリシーは、アプリケーション「Miru」（Android版のほか、今後提供される Windows / macOS / iOS 等の各プラットフォーム版を含みます。以下「本アプリ」）における利用者情報の取り扱いについて定めるものです。本アプリは個人開発者（以下「運営者」）が開発・提供しています。

プラットフォームによって利用できる機能（音声入力・カレンダー取り込みなど）は異なる場合があります。本ポリシーのうち特定の機能に関する記載は、その機能が提供されているプラットフォームにのみ適用されます。

1. 基本方針

本アプリは、マインドマップ・ノート作成アプリです。作成されたデータは原則としてお使いの端末内にのみ保存されます。クラウド同期・AI機能など、外部へのデータ送信を伴う機能は、いずれも利用者が明示的に操作した場合にのみ動作します。

運営者は、広告SDK・アクセス解析SDKを本アプリに組み込んでいません。利用者の行動追跡や、広告目的のデータ収集は一切行いません。

本アプリは、機能の利用状況（機能ごとの利用回数・利用の時間帯・曜日）を、個人を特定できない形の集計データとして記録することがあります。この集計データは、ノートの内容やアカウント情報とは結び付けられず、特定の利用者を識別することはできません（詳細は「2.7」を参照）。

2. 取り扱う情報

2.1 利用者が作成する情報（コンテンツデータ）

  • ノート・キャンバスの内容（テキスト、手書きストローク、レイアウト、タグ、期限など）
  • 添付ファイル（画像、ファイル、動画への参照）
  • クイックメモ
  • AI機能の対話履歴、AIが生成した問題集データ
  • パーソナライズ機能の設定・学習データ

これらは端末内に保存され、後述のクラウド同期またはAI機能を利用した場合に限り外部へ送信されます。

2.2 アカウント情報（クラウド同期を利用する場合のみ）

クラウド同期のために Google アカウントでサインインした場合、以下を取得します。

  • Google アカウントのメールアドレス・表示名・ユーザー識別子（UID）
  • 端末の識別子（ANDROID_ID）および機種名 — 複数端末からの同時編集を検知し、データの上書き事故を防ぐ目的でのみ使用します

2.3 カレンダー情報（予定取り込み機能を利用する場合のみ）

「カレンダー取り込み」機能の利用時に、端末のカレンダー予定を読み取ります（Android版では READ_CALENDAR 権限）。読み取りは利用者が取り込み画面を開いたときにのみ行われ、利用者が選択した予定だけがタスクとして本アプリ内に保存されます。カレンダー情報が自動的に外部へ送信されることはありません。

2.4 音声（AI音声入力を利用する場合のみ）

AI音声入力の利用時に、マイクから音声を録音します（Android版では RECORD_AUDIO 権限）。録音された音声は文字起こしのため後述のAI処理基盤へ送信され、処理後にサーバーへ保存されることはありません。録音は利用者が録音ボタンを操作している間にのみ行われます。

2.5 購入情報

有料機能の購入・購読は、Google Play をはじめとする各プラットフォームが提供する決済システムを通じて行われます。クレジットカード番号などの決済情報を運営者が取得することはありません。本アプリは購入状態（購入済みかどうか）のみを端末内に保存します。

2.6 取得しない情報

本アプリは、位置情報・連絡先・通話履歴・広告ID・カメラを取得しません。

2.7 匿名の利用統計

本アプリは、プロダクト改善のため、以下の情報を個人を特定できない形の集計データとして記録することがあります。

  • どの機能が、1日あたり何回使われたか（機能名と回数のみ。ノートの内容や検索語などは含まれません）
  • 利用の時間帯（0〜23時）・曜日ごとの利用回数

この記録には、氏名・メールアドレス・アカウントID・端末IDなど、利用者を特定できる情報を一切含みません。データは端末内で1日分だけ集計したうえで、日ごとに使い捨てられるランダムな値（当日限りのハッシュ値）とともにサーバーへ送信されます。このハッシュ値は「その日、何人が利用したか」を数える目的のみに使われ、日をまたいで同一人物を追跡することはできない設計です。サーバー側では、送信された回数を集計カウンタに加算するのみで、個別の利用者ごとの記録は保持しません。

3. 利用目的

取得した情報は、以下の目的にのみ利用します。

  1. 本アプリの機能提供（ノート作成・同期・AI処理・リマインダー通知など）
  2. 複数端末間でのデータ整合性の維持（同時編集の検知）
  3. 有料機能の提供状態の管理
  4. 不具合対応・お問い合わせ対応

4. クラウド同期におけるデータの取り扱い

クラウド同期（バックアップ／復元・自動同期）を利用した場合、コンテンツデータは Google LLC が提供する Firebase（Cloud Firestore / Cloud Storage）に保存されます。

  • アップロードされるデータは、送信前に AES-256-GCM 方式で暗号化され、暗号化された状態で保存されます。
  • 本暗号化は「保存時・通信時の暗号化」であり、ゼロ知識（エンドツーエンド）暗号化ではありません。複数端末での復元を可能にするため、暗号鍵は利用者の Google アカウントに紐付けてクラウド上で管理されます。このため、システムの管理権限を持つ運営者は技術的にはデータを復号し得る立場にありますが、利用者のデータ内容を閲覧することはありません。
  • クラウド同期を利用しない場合、コンテンツデータが Firebase に保存されることはありません。

5. AI機能におけるデータの取り扱い

AI機能（要約・タスク分解・マインドマップ生成・対話・音声入力・画像からのテキスト抽出など）を利用した場合、その処理に必要な範囲のテキスト・画像・音声が、運営者のサーバー（Firebase Cloud Functions、東京リージョン）を経由して Google LLC の Gemini API に送信されます。

  • 送信は、利用者がAI機能を明示的に実行したときにのみ行われます。
  • 運営者のサーバーは、送信された本文・画像・音声を保存しません。障害対応・コスト管理のため、データ量やトークン数などの統計的なメタデータのみを記録します。
  • Gemini API は有料プラン（Paid Services）で利用しており、Google の利用規約に基づき、送信されたデータが Google の AI モデルの学習・改善に利用されることはありません。
  • なお、手書き文字認識・画像内テキスト認識の一部（ML Kit）は端末内で処理され、外部に送信されません。

6. YouTube 動画の埋め込みについて

ノートに YouTube 動画を添付し再生する場合、YouTube の埋め込みプレイヤーを使用します。再生時には YouTube（Google LLC）との間で通信が発生し、Google のプライバシーポリシーおよび YouTube 利用規約が適用されます。動画を再生しない限り、この通信は発生しません。

7. 第三者への提供

運営者は、利用者の情報を第三者に販売・提供しません。ただし、以下の場合を除きます。

  1. 上記の機能提供に必要な範囲での処理の委託（Google LLC — Firebase / Gemini API。同期データは日本国内（東京リージョン、asia-northeast1）の Google のデータセンターに保存されます）
  2. 法令に基づく開示請求があった場合
  3. 人の生命・身体・財産の保護のために必要で、本人の同意を得ることが困難な場合

8. データの保持期間と削除

  • 端末内のデータ: アプリのアンインストールにより削除されます。
  • クラウド上の同期データ: 利用者が同期をやめた後も、復元に備えてサーバー上に保持されます。
  • アカウントとクラウドデータの削除: アプリ内の同期設定画面にある「アカウントとクラウドデータを削除」から、いつでもご自身で完全に削除できます（クラウド上の全データ・暗号鍵・アカウントが即時に削除されます）。アプリをすでにアンインストールした場合など、アプリ内から操作できない場合は、下記のお問い合わせ先までご連絡ください。確認のうえ、合理的な期間内（原則30日以内）に削除します。

9. 安全管理措置

  • クラウドとの通信はすべて TLS により暗号化されます。
  • クラウドに保存されるコンテンツデータは前述のとおり AES-256-GCM で暗号化されます。
  • クラウド上のデータへのアクセスは、Firebase の認証・セキュリティルールにより本人のアカウントに制限されています。

10. 子どものプライバシー

本アプリは特定の年齢層を対象としたものではなく、13歳未満の子どもから意図的に個人情報を収集することはありません。

11. 本ポリシーの変更

法令の変更や機能の追加に伴い、本ポリシーを改定することがあります。重要な変更がある場合は、アプリ内またはストアページ上でお知らせします。最新版は常に本ページに掲載されます。
